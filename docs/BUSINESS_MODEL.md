## 1. 시장 분석

### 1-1. 국내 노인장기요양 시장 규모 및 성장 추이

한국은 세계에서 가장 빠르게 고령화가 진행되는 국가 중 하나로, 2025년 65세 이상 인구가 전체의 20%를 초과하여 **초고령사회**에 진입했습니다. 이에 따라 노인장기요양보험 지출 및 관련 시장이 급격히 성장하고 있습니다.

| 연도 | 장기요양보험 급여 지출(조원) | 수급자 수(만 명) | 전년 대비 성장률 |
|------|---------------------------|-----------------|----------------|
| 2024 | 약 12.5 | 약 110 | 8.5% |
| 2025 | 약 13.8 | 약 120 | 10.4% |
| 2026(E) | 약 15.5 | 약 132 | 12.3% |
| 2027(E) | 약 17.5 | 약 145 | 12.9% |
| 2028(E) | 약 20.0 | 약 160 | 14.3% |
| 2029(E) | 약 22.5 | 약 175 | 12.5% |
| 2030(E) | 약 25.0 | 약 190 | 11.1% |

> **핵심 포인트**: 2030년까지 장기요양 시장은 연평균 **10-15%** 성장이 예상되며, 수급자 수는 약 190만 명에 이를 전망입니다.

#### 성장 동인

- **인구 고령화 가속**: 베이비부머 세대(1955-1963년생)의 본격적 고령 진입
- **장기요양 등급 확대**: 인지지원등급 신설 등 수급 범위 확대 정책
- **1인 고령가구 증가**: 독거노인 비율 증가로 시설·재가 서비스 수요 급증
- **정부 정책 지원**: 커뮤니티 케어(지역사회 통합돌봄) 정책 확대

### 1-3. 디지털 전환 현황 및 기회

#### 현재 상황 (Pain Point)

| 영역 | 현황 | 문제점 |
|------|------|--------|
| **시설-보호자 소통** | 전화, 카카오톡 개인 채팅 | 기록 미보존, 비체계적, 담당자 변경 시 히스토리 유실 |
| **일과 기록** | 수기 작성 또는 엑셀 | 실시간성 부족, 보호자 접근 불가 |
| **식단/투약 관리** | 종이 게시판, 구두 전달 | 오류 가능성 높음, 추적 불가 |
| **송영 관리** | 전화 연락, 수기 명부 | 실시간 위치 파악 불가, 보호자 불안 |
| **급여 청구** | 국민건강보험공단 시스템 수동 입력 | 이중 작업, 오류 발생 빈도 높음 |
| **보호자 만족도** | 비정기적 전화 또는 월 1회 가정통신문 | 정보 비대칭, 불신 발생 |

#### 디지털 전환 기회

```
┌─────────────────────────────────────────────────────────┐
│                   디지털 전환 기회 맵                      │
├──────────────────┬──────────────────────────────────────┤
│  즉시 도입 가능    │  ● 실시간 소통 플랫폼                  │
│  (1년 이내)       │  ● 디지털 일과 기록                    │
│                  │  ● 전자 식단표/투약 관리                │
├──────────────────┼──────────────────────────────────────┤
│  단기 도입 가능    │  ● 송영 차량 실시간 추적                │
│  (1-2년)         │  ● 급여 청구 자동화                    │
│                  │  ● 보호자 앱 연동                     │
├──────────────────┼──────────────────────────────────────┤
│  중장기 도입      │  ● AI 기반 건강 변화 예측               │
│  (2-3년)         │  ● IoT 센서 연동 (활동량, 수면 등)      │
│                  │  ● 빅데이터 분석 기반 케어 최적화         │
└──────────────────┴──────────────────────────────────────┘
```

> **시장 기회**: 전체 시설의 80% 이상이 아직 체계적인 디지털 솔루션을 도입하지 않은 **블루오션** 영역이며, 특히 **시설-보호자 간 실시간 정보 동기화**를 제공하는 전문 솔루션은 사실상 부재합니다.

## 2. 경쟁사 분석

### 2-1. 주요 경쟁사 비교

| 구분 | **케어포(CareFor)** | **케어닥(CareDoc)** | **돌봄플러스** | **SilverSync** |
|------|-------------------|-------------------|-------------|---------------|
| **핵심 기능** | 급여 계산/청구 | 방문요양 매칭 플랫폼 | 시설 행정 관리 | 시설 생활 전반 + 실시간 보호자 연결 |
| **타겟** | 요양기관 행정팀 | 요양보호사/이용자 | 시설 관리자 | 시설 + 보호자 (양면 플랫폼) |
| **주요 가치** | 급여 계산 정확성, 청구 자동화 | 요양보호사 매칭, 후기 기반 신뢰 | 입퇴소 관리, 문서 관리 | 시설-보호자 실시간 소통, 일과 공유 |
| **보호자 기능** | 없음 | 제한적 (매칭 후기) | 없음 | **핵심 기능 (전용 앱)** |
| **실시간성** | 낮음 (월 단위 정산) | 중간 (매칭 알림) | 낮음 (행정 위주) | **높음 (실시간 알림/공유)** |
| **가격대** | 월 5-15만원 | 중개 수수료 기반 | 월 3-10만원 | 월 3-15만원 |
| **강점** | 급여 계산 특화, 공단 연동 | 높은 인지도, 매칭 노하우 | 행정 효율화 | 보호자 경험 혁신, 양면 가치 |
| **약점** | 소통 기능 부재 | 시설 운영 기능 없음 | 보호자 기능 부재 | 신규 진입 (인지도 부족) |

### 2-3. SilverSync 핵심 차별점

| 차별점 | 설명 | 경쟁사 대비 우위 |
|--------|------|----------------|
| **양면 플랫폼** | 시설용 + 보호자용 앱 동시 제공 | 경쟁사는 대부분 한쪽만 타겟 |
| **실시간 일과 공유** | 식사, 활동, 건강 상태 실시간 업데이트 | 보호자가 매일 확인 가능 |
| **송영 알리미** | 차량 실시간 위치, 탑승/하차 알림 | 보호자 안심 기능 (경쟁사 미보유) |
| **AI 건강 인사이트** | 활동 패턴 변화, 이상 징후 조기 감지 | 데이터 기반 예방적 케어 |
| **통합 솔루션** | 소통 + 행정 + 급여를 하나의 플랫폼에서 | 별도 시스템 간 전환 불필요 |
| **보호자 만족도 향상** | 시설 평판/재이용률 직접적 개선 | 시설의 영업력 강화 도구 |

### 2-4. 진입 장벽 및 해자(Moat) 구축 전략

| 해자 유형 | 전략 | 설명 |
|----------|------|------|
| **네트워크 효과** | 시설-보호자 양면 네트워크 | 시설이 많을수록 보호자 유입, 보호자가 많을수록 시설 가입 촉진 |
| **전환 비용** | 데이터 축적 + 워크플로 통합 | 축적된 어르신 기록, 커뮤니케이션 히스토리가 전환 장벽 형성 |
| **데이터 우위** | 시설별 운영 데이터 축적 | AI 모델 학습 데이터 확보로 기능 고도화 |
| **규제 대응** | 장기요양 급여 청구 연동 | 공단 시스템 연동 인증/노하우 축적 |

### 3-2. 1차 타겟: 주간보호센터

| 항목 | 내용 |
|------|------|
| **전국 개소 수** | 약 5,000개소 |
| **평균 이용 어르신** | 20-40명/시설 |
| **1차 타겟 선정 이유** | 매일 등·하원 → 보호자와 소통 빈도 높음, 송영 관리 필수, 시설 규모가 적당하여 빠른 도입 가능 |
| **핵심 Pain Point** | 보호자 전화 응대 과다 (하루 10-30건), 송영 차량 위치 문의, 일과 보고 수기 작성 부담 |
| **의사결정자** | 센터장 (대부분 사회복지사 1급, 40-50대) |
| **예상 구매력** | 월 3-15만원 (정부 보조 + 자체 예산) |
| **도입 장벽** | 디지털 리터러시 편차, 비용 부담, 기존 업무 방식 변경 저항 |

#### 주간보호센터 페르소나

> **"김센터장" (52세, 여성, 주간보호센터 운영 8년차)**
> - 매일 아침 7시 출근, 송영 차량 운행 확인
> - 하루 15-20통의 보호자 전화 응대 (주로 "오늘 뭐 드셨어요?", "약 잘 드셨어요?")
> - 월말마다 3일간 밤새 급여 청구서 작성
> - "보호자님들이 걱정되시는 건 이해하지만, 전화 받느라 어르신 돌봄에 집중하기 어려워요"

### 3-3. 2차 타겟: 요양원/요양병원

| 항목 | 내용 |
|------|------|
| **전국 개소 수** | 요양원 약 4,200개 + 요양병원 약 1,500개 |
| **평균 이용 어르신** | 30-100명/시설 |
| **2차 타겟 선정 이유** | 입소형이라 보호자 불안감 더 큼, 24시간 케어 기록 필요, 높은 객단가 가능 |
| **핵심 Pain Point** | 보호자 면회 제한 시 소통 수단 부재, 야간 상태 보고 어려움, 시설 신뢰도 관리 |
| **의사결정자** | 시설장/원장 (간호사 또는 사회복지사 출신) |
| **예상 구매력** | 월 8-20만원 |

### 3-4. 3차 타겟: 방문요양 서비스

| 항목 | 내용 |
|------|------|
| **전국 개소 수** | 약 15,000개소 |
| **주요 인력** | 요양보호사 약 50만 명 |
| **3차 타겟 선정 이유** | 시장 규모 최대, 다만 분산형 구조로 플랫폼 도입 난이도 높음 |
| **핵심 Pain Point** | 요양보호사-보호자 간 소통 비체계적, 서비스 품질 관리 어려움, 근무 기록 검증 |
| **의사결정자** | 기관 대표 (소규모 기관 다수) |
| **예상 구매력** | 월 3-8만원 (소규모 기관 다수) |

### 3-5. 보호자 (B2C 사이드)

| 항목 | 내용 |
|------|------|
| **대상** | 고령 부모를 둔 40-60대 자녀 |
| **추정 규모** | 장기요양 수급자 가족 약 100만+ 세대 |
| **핵심 니즈** | 부모 일상 확인, 안부 걱정 해소, 투약/식사 관리 확인, 시설과 원활한 소통 |
| **디지털 역량** | 높음 (스마트폰 활용 능숙, 카카오톡/네이버 앱 사용) |
| **지불 의향** | 무료 기본 / 프리미엄 기능(AI 리포트 등)에 월 5,000-10,000원 가능 |

#### 보호자 페르소나

> **"이주현" (48세, 남성, 직장인, 서울 거주)**
> - 지방에 계신 어머니(78세)가 주간보호센터 이용 중
> - 주 1회 전화로 안부 확인, 월 1회 방문
> - "엄마가 오늘 뭐 하셨는지, 밥은 잘 드셨는지 매일 알고 싶어요"
> - "센터에 전화하면 바쁘실까 봐 미안하고, 안 하면 걱정되고..."

## 4. 구독 플랜 설계

### 4-1. 플랜 요약

| 구분 | **Basic** | **Standard** | **Premium** | **Enterprise** |
|------|-----------|-------------|------------|---------------|
| **월 요금** | **3만원**/시설 | **8만원**/시설 | **15만원**/시설 | **별도 협의** |
| **연간 결제 시** | 2.5만원/월 (17% 할인) | 6.5만원/월 (19% 할인) | 12만원/월 (20% 할인) | 별도 협의 |
| **어르신 수** | 30명까지 | 50명까지 | **무제한** | 무제한 |
| **관리자 계정** | 2개 | 5개 | 10개 | 무제한 |
